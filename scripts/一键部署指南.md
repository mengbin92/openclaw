## 1. 概述

一键部署脚本是我自己编写的自动化安装工具，可以帮助新手用户快速完成 OpenClaw 的完整部署。脚本会自动检测操作系统、安装必要的依赖、配置 GPUNexus 模型，并生成配置文件。

**为什么使用一键部署脚本？**

- **简单快捷**：一行命令完成全部部署流程
- **自动检测**：智能识别操作系统，自动选择合适的安装方式
- **依赖完备**：自动安装 Node.js 及编译所需的系统依赖
- **配置完善**：自动生成配置文件，支持多种模型选择

**前置要求：**

- 拥有一个 GPUNexus 平台的 API Key（获取方式参考 [使用手册.md](./使用手册.md)）
- 能够执行 Shell 脚本（Linux/macOS/WSL）或 PowerShell 脚本（Windows）

## 2. 快速开始

### 2.1 Linux/macOS/WSL 用户

```bash
# 方式一：交互模式（推荐）
curl -fsSL https://raw.githubusercontent.com/mengbin92/openclaw/main/scripts/install-openclaw.sh | bash

# 方式二：使用参数直接部署
curl -fsSL https://raw.githubusercontent.com/mengbin92/openclaw/main/scripts/install-openclaw.sh | bash -s -- -k "sk-xxxxxxxxxxxxx" -m 2
```

**或者下载脚本到本地执行：**

```bash
# 下载脚本
curl -fsSL -o install-openclaw.sh https://raw.githubusercontent.com/mengbin92/openclaw/main/scripts/install-openclaw.sh

# 添加执行权限
chmod +x install-openclaw.sh

# 运行脚本
./install-openclaw.sh
```

### 2.2 Windows 用户

```powershell
# 方式一：下载脚本到本地执行
# 1. 访问 https://raw.githubusercontent.com/mengbin92/openclaw/main/scripts/install-openclaw.ps1
# 2. 保存为 install-openclaw.ps1
# 3. 以管理员身份打开 PowerShell

# 运行脚本
.\install-openclaw.ps1

# 方式二：使用参数直接部署
.\install-openclaw.ps1 -ApiKey "sk-xxxxxxxxxxxxx" -ModelChoice 2
```

> **注意：** Windows 用户需要先安装 Node.js（https://nodejs.org）和 Git（https://git-scm.com）。如果未安装，脚本会尝试通过 Chocolatey 自动安装。

## 3. 脚本功能

### 3.1 支持的操作系统

| 脚本 | 支持的系统 |
| ---- | ---------- |
| `install-openclaw.sh` | macOS、Linux (Ubuntu/Debian、CentOS/RHEL、Arch)、WSL、Git Bash |
| `install-openclaw.ps1` | Windows 10/11 |

**Linux 发行版支持：**

- **Debian 系列**：Ubuntu、Debian、Linux Mint、Pop!_OS
- **RHEL 系列**：CentOS、RHEL、Fedora、Rocky Linux、AlmaLinux
- **Arch 系列**：Arch Linux、Manjaro、EndeavourOS

### 3.2 自动安装内容

脚本会自动完成以下内容的安装和配置：

1. **系统依赖**
   - build-essential、cmake、gcc、g++、make（Linux 编译依赖）
   - git、jq、curl、python3

2. **Node.js**
   - 自动检测 Node.js 是否已安装
   - 未安装时自动安装 LTS 版本
   - 支持通过 nvm 管理 Node.js 版本

3. **OpenClaw**
   - 全局安装 `openclaw` npm 包
   - 自动配置 npm 镜像源（使用 npmmirror）
   - 配置 Git 使用 HTTPS 协议（解决 SSH 密钥问题）

4. **配置文件**
   - 自动生成 `~/.openclaw/openclaw.json`
   - 自动配置 GPUNexus 模型提供方
   - 自动设置默认模型

### 3.3 支持的模型

| 序号 | 模型名称 | API 接口 | 说明 |
| ---- | -------- | -------- | ---- |
| 1 | MiniMax-M2.1 | OpenAI 兼容接口 | 使用 OpenAI 格式调用 |
| 2 | GPUNexus | Claude Code 接口 | 使用 Anthropic 消息格式 |

## 4. 命令行参数

示例:
    .\install-openclaw.ps1 -ApiKey "sk-xxxxxxxxxxxxx" -ModelChoice 1
    .\install-openclaw.ps1 -ApiKey "sk-xxxxxxxxxxxxx"

## 7. 验证部署

### 7.1 查看配置文件

部署完成后，配置文件位于 `~/.openclaw/openclaw.json`。

**查看配置：**

```bash
# Linux/macOS
cat ~/.openclaw/openclaw.json

# Windows
type %USERPROFILE%\.openclaw\openclaw.json
```

**关键配置项：**

| 字段 | 说明 |
| ---- | ---- |
| `models.providers` | 模型提供方配置 |
| `agents.defaults.model.primary` | 默认使用的模型 |
| `gateway.port` | 服务端口（默认 18789） |
| `gateway.auth.token` | 访问令牌 |

### 7.2 启动服务

```bash
# 启动服务
openclaw start

# 查看服务状态
openclaw status

# 重启服务
openclaw restart
```

### 7.3 访问 WebUI

服务启动后，在浏览器中访问：

```
http://127.0.0.1:18789/?token=YOUR_TOKEN
```

> **注意：** 将 `YOUR_TOKEN` 替换为配置文件中的 `gateway.auth.token` 值。

## 8. 常见问题

### Q1: 脚本执行失败怎么办？

1. 确认网络连接正常
2. 检查是否有足够的磁盘空间
3. Linux 用户确保有 sudo 权限
4. 查看终端输出的错误信息

### Q2: Node.js 安装失败怎么办？

**macOS 用户：**

```bash
# 安装 Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 使用 Homebrew 安装 Node.js
brew install node
```

**Linux 用户：**

```bash
# 手动安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Q3: 安装过程中断如何恢复？

```bash
# 重新运行脚本即可，脚本会自动检测已安装的组件并跳过
./install-openclaw.sh
```

### Q4: 如何更新 API Key？

直接编辑配置文件中的 `apiKey` 字段，然后重启服务：

```bash
# 编辑配置
nano ~/.openclaw/openclaw.json

# 重启服务
openclaw restart
```

### Q5: 内存不足导致安装失败？

Linux 脚本会自动检测内存情况，如果内存小于 2GB 且 swap 小于 2GB，会自动添加 4GB swap 空间。

如需手动添加：

```bash
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### Q6: Windows PowerShell 无法运行脚本？

1. 确保以管理员身份运行 PowerShell
2. 检查是否启用了脚本执行策略：

```powershell
# 查看执行策略
Get-ExecutionPolicy

# 如果受限，设置为允许
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Q7: 如何卸载 OpenClaw？

```bash
# 卸载 npm 包
npm uninstall -g openclaw

# 删除配置文件（可选）
rm -rf ~/.openclaw
```
