# OpenClaw 接入飞书机器人操作指南

> OpenClaw（原 Clawdbot，昵称龙虾）是开源、本地优先的 AI 代理网关，能让大模型在服务器/本地电脑 7×24 小时运行，支持操作电脑、执行命令、浏览网页等能力。将其接入企业飞书后，可实现智能对话、消息推送、收发文件、审批交互、数据同步等自动化办公场景。

本教程从环境准备到功能测试，手把手完成全流程对接。

## 1. 前置准备

在开始配置之前，请确保：

| 准备项 | 说明 |
|-------|------|
| 飞书管理员账号 | 具备自建应用/机器人创建、权限配置、发布权限 |
| Node.js 环境 | 本地/服务器已安装 Node.js 16.x 及以上，npm/pnpm 可用 |
| 网络环境 | 服务器部署保证网络通畅（飞书 WebSocket 长连接无需公网域名） |
| 飞书开放平台 | 账号完成企业认证，客户端为最新版 |

## 2. 步骤一：安装 OpenClaw 核心程序

### 2.1 安装命令

```bash
# npm 国内镜像加速安装
npm install -g openclaw@latest --registry=https://registry.npmmirror.com

# 或使用 pnpm 安装
pnpm add -g openclaw@latest
```

### 2.2 验证安装

```bash
openclaw -v
```

执行后终端显示版本号，即为安装成功。

---

## 3. 步骤二：安装并启用飞书官方插件

### 3.1 启用内置插件

```bash
# 启用内置飞书插件
openclaw plugins enable feishu

# 查看插件状态
openclaw plugins list
```

### 3.2 手动安装（若未内置）

```bash
openclaw plugins install @openclaw/feishu
```

### 3.3 验证成功

验证成功会显示：
```
Feishu feishu loaded
```

## 4. 步骤三：在飞书开放平台创建企业自建应用

### 4.1 创建应用

1. 打开 [飞书开放平台](https://open.feishu.cn/app)
2. 使用管理员账号登录
3. 点击**创建企业自建应用**

![飞书开放平台](https://oss.gpunexus.com/doc/openclaw/feishu/2.png)

### 4.2 填写应用信息

- **名称**：如 `OpenClaw AI 助手`
- **描述**：应用功能说明
- **图标**：上传应用图标

### 4.3 获取凭证

进入应用详情页，在**凭证与基础信息**中复制：
- **App ID**
- **App Secret**

> **重要**：务必保存 App ID 和 App Secret，这是后续配置的核心凭证。

### 4.4 添加机器人能力

1. 左侧导航：**应用能力** > **添加应用能力**
2. 选择**机器人**
3. 点击**添加**，开启机器人核心功能

![添加机器人能力](https://oss.gpunexus.com/doc/openclaw/feishu/4.png)

## 5. 步骤四：配置 OpenClaw 飞书通道

### 5.1 交互式命令行配置

执行通道添加命令进入交互配置：

```bash
openclaw channels add
```

按提示依次配置：

| 配置项 | 选择/输入 |
|-------|----------|
| 通道类型 | Feishu/Lark (飞书) |
| 认证方式 | Enter App Secret |
| App Secret | 粘贴飞书应用凭证 |
| App ID | 粘贴飞书应用凭证 |
| 连接模式 | WebSocket (default) |
| 域名 | Feishu (feishu.cn) - China |
| 群聊策略 | Open - respond in all groups (requires mention) |
| 完成 | Finished |
| 绑定代理 | Yes，绑定到默认代理 main |

### 5.2 配置文件方式（批量部署）

创建配置文件：

```yaml
channels:
  feishu:
    enabled: true
    appId: "你的App ID"
    appSecret: "你的App Secret"
    connectionMode: "websocket"
    domain: "feishu.cn"
    groupChatPolicy: "open"
```

保存后执行：

```bash
openclaw gateway restart
```

## 6. 步骤五：飞书自建应用配置（权限 + 事件订阅）

### 6.1 配置权限管理

1. 左侧导航：**开发配置** > **权限管理**
2. 点击**开通权限**

![权限管理入口](https://oss.gpunexus.com/doc/openclaw/feishu/5.png)

### 6.2 开通即时通信权限

1. 选择**应用身份权限（tenant_access_token）**
2. 在分类中选择**消息与群组**
3. 勾选以下核心权限：

![权限选择](https://oss.gpunexus.com/doc/openclaw/feishu/6.png)

| 权限名称 | 权限标识 | 说明 |
|---------|---------|------|
| 获取与更新群组信息 | `im:chat` | 获取群组基本信息 |
| 创建群 | `im:chat:create` | 创建新的群组 |
| 解散群 | `im:chat:delete` | 解散已有群组 |
| 操作群发言权限 | `im:chat:moderation:write_only` | 管理群发言权限 |

4. 点击**确认开通权限**

> **提示**：企业内部应用免审直接开通。

### 6.3 配置 WebSocket 长连接事件订阅

1. 左侧导航：**事件与回调** > **事件配置**
2. 订阅方式选择：**使用长连接接收事件（WebSocket）**
3. 点击**保存**
4. 点击**添加事件**，依次添加：

| 事件 | 说明 |
|-----|------|
| `im.message.receive_v1` | 接收消息 |
| `im.message.message_read_v1` | 已读回执 |
| `im.chat.member.bot.added_v1` | 机器人进群 |
| `im.chat.member.bot.deleted_v1` | 机器人被移出群 |

5. 所有事件选择**应用身份订阅**

## 8. 步骤七：启动 OpenClaw 网关服务

### 8.1 启动命令

```bash
# 默认端口启动
openclaw gateway

# 自定义端口（避免冲突）
openclaw gateway --port 8888

# 服务器后台运行（pm2）
pm2 start openclaw -- gateway
```

### 8.2 验证启动成功

启动成功终端显示：
```
Gateway running on http://0.0.0.0:端口号
```

飞书通道状态为 `Running/Connected`。

## 9. 步骤八：飞书端添加机器人并配对测试

### 9.1 添加机器人

1. 飞书客户端搜索机器人名称（如 `OpenClaw AI 助手`）
2. 进入单聊

### 9.2 完成配对

1. 发送消息，获取**配对码**
2. 终端执行配对命令：

```bash
openclaw pairing approve feishu 配对码
```

### 9.3 功能测试

**单聊测试**：
- 发送「你好」，机器人回复即成功

**群聊测试**：
- @机器人 发送「介绍下自己」
- 机器人回复即成功

**文件收发测试**：
- 发送一张图片/文档给机器人
- 确认能接收并回复

## 10. 常见问题排查

### 问题1：机器人无响应

**排查步骤**：
1. 检查网关状态：`openclaw gateway status`
2. 核对飞书 App ID/Secret 与配置是否一致
3. 确认飞书应用已发布（状态显示「已启用」）

### 问题2：长连接失败

**排查步骤**：
1. 检查飞书事件配置页
2. 确认选中「WebSocket 长连接」而非 HTTP 回调

### 问题3：插件启用失败

**解决方案**：
```bash
# 升级 OpenClaw
npm update -g openclaw@latest

# 删除旧插件后重装
rm -rf ~/.openclaw/extensions/feishu
openclaw plugins install @openclaw/feishu
```

### 问题4：无法收发文件

**排查步骤**：
1. 检查飞书权限配置
2. 确认已开通 `aily:file:read` 和 `aily:file:write` 权限

## 11. 企业级拓展与优化

### 多机器人配置

```bash
# 添加多个飞书通道，绑定不同 AI 代理
openclaw channels add
```

### 权限精细化

将群聊策略改为 `Allowlist`，仅开放核心业务群：

```yaml
channels:
  feishu:
    groupChatPolicy: "allowlist"
    allowedGroups:
      - "群ID1"
      - "群ID2"
```

### 高可用部署

```bash
# pm2 守护 + 监控告警
pm2 start openclaw --name "openclaw-gateway" -- gateway
pm2 save
pm2 startup
```

### 飞书生态整合

可进一步对接：
- **飞书文档**：自动生成会议纪要
- **多维表格**：任务同步
- **日历**：日程管理
